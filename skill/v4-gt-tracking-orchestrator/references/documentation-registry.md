# Registro de Documentacao Oficial

Usar `documentation-registry.json` como indice. Antes de qualquer implementacao:

1. abrir a documentacao oficial;
2. confirmar que a pagina ainda existe;
3. confirmar produto, versao, conta, regiao e elegibilidade;
4. registrar `consulted_at` e regra relevante;
5. atualizar o registro local quando a URL mudar;
6. nao copiar documentacao inteira para a Skill;
7. preferir guias oficiais, API reference e changelog;
8. usar fontes secundarias somente para diagnostico complementar.

## Plataformas cobertas

- OpenAI/ChatGPT/Codex e Skills;
- Kommo, RD Station, HubSpot, Pipedrive, ActiveCampaign, Salesforce, Zoho e Moskit;
- Meta, Google Ads, TikTok, LinkedIn e Microsoft Advertising;
- GTM, GA4 e server-side tagging;
- WhatsApp Cloud API, Twilio e Evolution;
- n8n e Make;
- Google Sheets, BigQuery, Supabase/Postgres;
- Shopify, WooCommerce e Stripe.

## Regra de atualizacao

URLs sao pontos de entrada, nao garantia de contrato eterno. O agente deve pesquisar dentro do dominio oficial quando a rota estiver quebrada ou quando precisar de endpoint/campo atual.
